# 开张 KAIZHANG · 产品介绍与 PRD 摘要

> 版本：v2.2  
> 形态：AI 产品学习工作台  
> 定位：帮助零基础用户沿「学 → 做 → 成」路径，做出并发布自己的第一个 AI 产品。

---

## 1. 产品一句话

**开张 KAIZHANG** 是一个像 AI coding 工具一样的学习工作台：用户从零开始沿学习路径读卡片、随时问「店小二」、把答案归档成笔记，最后拿工具箱资产做出产品，并发布到广场获得第一批反馈。

---

## 2. 目标用户

### 2.1 核心用户：零基础造物者

他们不是程序员，但被 AI coding 吸引：

- 想用 Cursor / Trae / Lovable 做东西
- 不知道 LLM、MCP、Agent、部署、数据库是什么意思
- 看教程容易放弃
- 做出来以后不知道怎么发布、怎么找第一批用户

### 2.2 次核心用户：已有产品但没有用户的人

他们已经能做出产品，但卡在：

- 没人用
- 不会宣发
- 不知道产品想法是否成立
- 缺少结构化反馈

---

## 3. 核心问题

AI coding 降低了“写代码”的门槛，但没有解决完整旅程：

```txt
不知道怎么学 → 不知道怎么做 → 不知道怎么上线 → 不知道怎么被人看到
```

市面上常见产品只解决其中一段：

- AI 工具：帮你写代码
- 导航站：给你链接
- 课程：教你概念
- 社区：让你发帖

**开张想把这条旅程焊成一个工作台。**

---

## 4. 产品结构

### 4.1 首页

用户打开后不是看到官网，而是看到一个工作台入口：

- 大输入框：直接问问题
- 预置按钮：零基础开始 / 解释概念 / 验证想法 / 怎么让人用
- 一步进入，不做复杂问卷

### 4.2 学习路径

学习路径是推荐主线，按顺序走：

| Phase | 名称 | 目标 |
|---|---|---|
| P0 | 认知与选型 | 知道 vibe coding 是什么，选对工具 |
| P1 | 基础概念扫盲 | 看懂 LLM / Token / Prompt / Agent 等术语 |
| P2 | 上手第一个产品 | 跑通第一个本地页面 |
| P3 | 功能拓展 | 数据库 / 登录 / 支付 |
| P4 | AI 增强开发 | MCP / Agent / Skills |
| P5 | 部署与上线 | 有公网地址 |
| P6 | 宣发与变现 | 找到用户和反馈 |

学习路径回答：**下一步该学什么**。

### 4.3 全部分组

全部分组是索引，不代表学习顺序。它按主题组织卡片：

- AI 基础概念
- 模型能力与选型
- Vibe Coding 工具
- MCP 与 Agent
- 前端
- 后端与数据库
- 部署与上线
- 宣发与变现

全部分组回答：**某个概念在哪查**。

### 4.4 店小二

原名 AI 副驾，现在更贴合产品气质，叫 **店小二**。

能力：

- 回答当前卡片问题
- 读取用户所在 Phase
- 读取当前正在看的卡片
- 读取已读卡片和最近笔记摘要
- 推荐相关卡片
- 支持图片、语音、文字输入
- 支持结构化回答卡片

核心价值：**它不是通用聊天框，而是知道你在开张里学到哪一步的陪跑助手。**

### 4.5 工具箱 Tools

工具箱不是讲解区，而是可拿走用的资产库。

内容包括：

- AgentSkills 标准包
- Prompt 模板
- Cursor Rules
- MCP 配置
- 起步模板
- 工作流配方
- Loop Engineering 模板
- Harness Engineering 模板

定位：**学习区负责理解，工具箱负责拿走使用。**

### 4.6 发布广场

发布广场是用户从“做出来”走向“被看到”的地方。

功能：

- 作品卡片流
- 作品详情页
- 点赞 / 收藏 / 评论
- 有用反馈赚积分
- 发布作品触发「开张大吉」仪式
- 生成开张喜报 PNG

### 4.7 我的空间

用户个人沉淀区：

- 我的笔记
- 我的作品
- 我的积分
- 设置

笔记默认以卡片网格展示，点击后进入详情。

## 5. 核心设计公理

### 5.1 资讯 × 咨询同源

知识卡片是可浏览态；店小二回答是可对话态。二者必须互通：

- 在卡片中选中文字 → 问店小二
- 店小二回答 → 推荐相关卡片
- 提问时自动附带当前卡片 / Phase

### 5.2 工作台，不是官网

界面不做营销大页，而是：

- 背景 + Mac 浮窗
- 三栏工作台
- 左栏路径 / 中栏内容 / 右栏店小二
- 深色 IDE 质感

### 5.3 学习必须导向行动

每个 Phase 都有动手检查点，不只是读文章。

### 5.4 沉淀有第二次生命

用户可以随时归档：

- 选中文本归档
- 店小二回答归档
- 手动新建笔记
- 语音记笔记

### 5.5 做完要被看到

发布广场和开张大吉仪式让作品从本地页面变成可传播内容。

## 8. 产品演示路径

推荐演示顺序：

1. 首页输入：「我零基础，想做一个 AI 工具」
2. 进入学习路径
3. 点 Phase 1 / Phase 4
4. 打开一张卡片，选中一句话问店小二
5. 店小二结构化回答，并推荐相关卡片
6. 归档为笔记
7. 打开工具箱，下载 AgentSkill
8. 发布广场发布作品
9. 触发开张大吉，保存喜报 PNG
10. 打开我的空间查看笔记和作品

## 9. 对外介绍文案

### 短版

开张 KAIZHANG 是一个 AI 产品学习工作台，帮零基础用户从看懂 AI coding，到做出第一个产品，再到发布作品获得反馈。

### 中版

开张 KAIZHANG 把学习路径、知识卡片、AI 店小二、可下载工具箱、笔记沉淀和发布广场放在一个工作台里。用户可以沿着「学 → 做 → 成」的路径，从零理解 AI coding，随时提问并归档，拿工具箱资产动手做，最后把作品发布到广场获得第一批反馈。

### 长版

AI coding 让写代码变简单，但对零基础用户来说，真正困难的是不知道从哪里开始、概念看不懂、工具不会选、部署不敢碰、做完也没人看到。开张 KAIZHANG 不是又一个工具导航站，而是一个完整的 AI 产品陪跑工作台：学习路径告诉你下一步学什么，知识卡片解释每个概念，店小二根据你的当前进度回答问题，工具箱提供可直接下载和使用的 Prompt、MCP、AgentSkills 等资产，笔记系统沉淀你的理解，发布广场让你的第一个产品被人看到。开张的目标不是让用户读完一堆教程，而是让他真的开张。
