## P-01 · 产品启动 prompt

**用途**：从一句话想法 → 完整可启动的 Next.js 项目

```
你是一名资深 Next.js 全栈工程师 + 产品经理。

我有一个产品想法：[一句话描述]

目标用户：[用户群]
核心场景：[他们什么时候用]
我想用的技术栈：Next.js 14 (App Router) + TypeScript + Tailwind + shadcn/ui + Supabase + Stripe

请帮我：
1. 用 3 行话给这个产品定一个清晰定位（替我做 elevator pitch）
2. 列出 MVP 必须有的 5 个核心功能（要冷酷，砍掉所有"nice to have"）
3. 设计 Supabase 数据表 schema（建表 SQL，含 RLS 策略思路）
4. 列出我需要装的 npm 包
5. 给我一个 30 分钟内能完成的"第一步行动清单"

不要给完整代码，给我"思路 + 决策"。我准备好开始动手时再来要代码。
```

**示例**：
> 我有一个产品想法：帮独立开发者自动写日报
> 目标用户：远程办公的独立开发者
> 核心场景：每天下午 5 点，AI 自动总结今天的 git commits + 日程，发邮件给 manager

## P-02 · 报错解决 prompt

**用途**：终端 / 浏览器报错时，让 AI 一步步排查

```
我遇到了下面的报错，请帮我系统排查：

```
[贴完整报错信息]
```

上下文：
- 我在做什么：[执行了什么命令 / 操作]
- 项目技术栈：[Next.js / Python / 等]
- 最近改了什么：[最近 1 小时内的改动]

请按以下顺序帮我：
1. 用一句话说这个错的"根本原因"
2. 给我 3 个最可能的具体原因（按概率排序）
3. 针对每个原因，告诉我"怎么验证"和"怎么修"
4. 如果信息不够你判断，问我具体要看哪个文件 / 跑哪个命令

不要给完整代码，先让我理解问题。
```

## P-03 · UI 描述 prompt

**用途**：把脑中的页面布局转成代码（不画图也能描述清楚）

```
请用 Next.js + Tailwind + shadcn/ui 实现下面的页面。

页面：[页面名，如"用户 dashboard 首页"]

整体布局：
- 顶部 header：左 logo "VibeMaker" / 右导航 Home · Docs · Pricing + 头像下拉
- 主体分两栏：左 sidebar 240px（菜单），右内容区
- 底部 footer：版权 + 社交链接

主体内容区：
- 一个"欢迎回来 [用户名]"标题
- 4 个统计卡片横向排列（总笔记 / 今日新增 / 收藏 / 已分享），每张有数字 + emoji + 趋势箭头
- 下方"最近笔记"列表（卡片网格 3 列），每张卡有 标题 / 摘要 / 日期 / 编辑按钮
- 右下角悬浮"+ 新建笔记"按钮

风格：
- 深色主题（默认 dark）
- 主色：indigo / blue 渐变
- 圆角 lg、阴影 sm、间距宽松
- 移动端单列响应式

要求：
- 用 Server Component 默认
- 数据用 mock（不用真接 DB）
- shadcn/ui 组件优先（Card / Avatar / DropdownMenu / Button）
- 给完整可运行的 app/dashboard/page.tsx + 必要的子组件
```

## P-04 · 功能迭代 prompt

**用途**：在已有代码上加功能，让 AI 不破坏已有结构

```
我要在现有项目里加一个功能。先按我的步骤来，不要一次性改太多。

要加的功能：[一句话描述]

需求细节：
- 用户操作流程：[第 1 步 → 第 2 步 → ...]
- 涉及的页面：[哪些页面要改]
- 涉及的数据：[需要新表 / 字段吗？]
- 边界情况：[空状态 / 错误处理]

执行步骤：
1. 先告诉我"为完成这个功能你打算改哪些文件、新建哪些文件"——给我列表，等我确认
2. 我说"开始"后，逐文件改：先改数据库 schema → 再改 API → 再改前端
3. 每改完一个文件，停下来告诉我"这文件做了什么、下一步是啥"
4. 最后给我"测试清单"：我应该手动测哪几个场景

约束：
- 严格遵守现有代码风格（看 .cursorrules）
- 不要删任何现有 API，要兼容
- 安全：用户输入都要服务端校验
```

## P-05 · 数据库设计 prompt

**用途**：从产品需求 → ER 图 + 建表 SQL

```
我要给我的产品设计数据库。

产品场景：[一句话描述]
核心实体：[列出实体，如"用户 / 笔记 / 标签 / 分享"]
核心场景：
- [场景 1，如"用户创建笔记"]
- [场景 2，如"用户给笔记加标签"]
- [场景 3，如"用户分享笔记给其他人，分享有过期时间"]

数据库：Supabase (Postgres)

请帮我：
1. 画 Mermaid erDiagram（用 ||--o{ 等关系符）
2. 写完整的建表 SQL，包括：
   - 主键用 uuid + gen_random_uuid()
   - 外键约束（含 on delete cascade）
   - created_at / updated_at 时间戳
   - 必要的索引
   - 用 enum 表示状态字段
3. 给每个表写一段 RLS（Row Level Security）策略——谁能读、谁能写
4. 列 3 个常见查询的 SQL 示例（不用 ORM，写 raw SQL）

输出按 (1)→(2)→(3)→(4) 顺序。
```

## P-06 · API 设计 prompt

**用途**：从产品功能 → RESTful 端点列表 + 请求/响应 schema

```
帮我为下面的功能设计 RESTful API。

功能：[功能名]
涉及的资源：[如"笔记 notes / 标签 tags"]
用户身份：[匿名 / 登录用户 / 管理员]

请输出：
1. 端点列表（5-8 个），格式：
   - GET /api/notes — 查询当前用户的笔记
   - POST /api/notes — 创建笔记
   - …

2. 对每个端点：
   - URL + HTTP 方法
   - 请求体 schema（TypeScript 类型）
   - 响应体 schema
   - 可能的状态码（200 / 400 / 401 / 403 / 404 / 500）+ 每种意义
   - 简短说明

3. 如何处理：
   - 鉴权（每个端点要不要登录）
   - 输入校验（用 zod 还是别的？给个示例 schema）
   - 错误响应格式（统一是 `{ error: string, code: string }` 吗）

技术栈：Next.js 14 App Router + Supabase Auth + zod。
```

## P-07 · 代码审查 prompt

**用途**：让 AI 帮你 review 一段代码（找 bug、性能、安全）

```
请帮我 review 下面这段代码，**像一个挑剔的资深工程师**。

```[language]
[贴代码]
```

请按下面 5 个维度审查：

1. **正确性**：有没有 bug、边界情况是否处理（null / undefined / 空数组 / 异常）
2. **安全**：有没有注入、XSS、CSRF、secret 泄露、权限绕过
3. **性能**：有没有 N+1、不必要的循环、内存泄漏、能加缓存的地方
4. **可维护性**：命名、注释、结构、能否抽函数
5. **风格**：是否符合现代最佳实践（async/await、Hooks、TS 严格模式等）

输出格式：
- 每条问题用"严重程度（🔴严重 / 🟡建议 / 🟢小细节）+ 位置 + 问题 + 建议改法"
- 最后给一个"如果只改 3 处，改哪 3 处"
```

## P-08 · 测试 prompt

**用途**：给已有函数 / 组件生成单元测试

```
请为下面这个函数 / 组件生成测试。

```[language]
[贴代码]
```

测试框架：[Jest / Vitest / Playwright]

测试要求：
1. 覆盖正常 happy path
2. 覆盖边界情况：[空输入 / null / 极大值 / 重复 / 不存在]
3. 覆盖异常路径：[网络错 / 数据库错 / 权限错]
4. 每个测试用 describe + it 结构，名字像"应该...当..."

输出：
- 完整可跑的测试文件
- 在文件顶部用注释列出"还缺什么测试"（你能想到但没写的）
```

## P-11 · Product Hunt 文案 prompt

**用途**：tagline + Maker Comment

```
我要在 Product Hunt 发布 [产品名]。

产品定位：[一句话]
为什么做：[创作背景]
最自豪的功能：[1-2 个]
未来计划：[1 句话]

请帮我：

1. **Tagline 5 版**（≤60 字符 英文，要 punchy）
   排序：最 punchy → 最 safe

2. **Description 文案**（≤260 字符 英文，PH 短描述）

3. **Maker Comment**（PH 第一条评论），结构：
   - 第一段：自我介绍 + 为什么做（讲故事）
   - 第二段：产品核心做什么（1-3 个亮点）
   - 第三段：免费的部分 + 如何使用
   - 第四段：我希望听到什么反馈
   - 第五段：感谢 + emoji
   总字数 ≈ 250 英文字

4. **预告贴**（X + Reddit r/SideProject）：上线前 24h 发的"明天 PH 见"短帖

风格：诚实 + 真诚，不要 hype 词（revolutionary / game-changing / 10x faster）。
```

## P-12 · X thread（BIP）prompt

**用途**：长 thread 分享教训 / 进度

```
帮我写一条 X thread，分享 vibe coding 实战教训。

主题：[一句话主题，如"我用 Cursor 30 天做了 3 个产品，学到 5 件事"]
我的角色：[独立开发者 / 学生 / 设计师转码]
具体细节：[列 3-5 条事实]

要求：
1. 首条（≤280 字符）—— 钩子句 + 数据 + emoji
2. 接下来 5-8 条（每条 ≤280 字符）—— 一个观点一条
3. 倒数第二条：复盘 / 反思
4. 最后一条：CTA（"关注我 @xxx 持续更新" / "产品链接 + 折扣码"）

风格：
- 第一人称、口语
- 每条独立成立（被零散转发也成立）
- 数据 / 截图位标 `[这里放截图]`
- 不要 "🧵 thread" 这种废话开头
```

## P-13 · 小红书笔记 prompt

**用途**：中文产品种草

```
帮我写一篇小红书种草笔记。

产品：[名字]
一句话定位：[…]
目标用户：[年龄 + 身份 + 痛点]
核心功能：[1-2 个核心]

笔记类型：[痛点解决 / 教程 / 对比 / 数据成绩 / 反向避雷] 任选一种

1. **标题**（≤20 字）—— 痛点 + 数字 + emoji + 钩子

## P-14 · 用户访谈 prompt（The Mom Test 风格）

**用途**：找用户验证想法时怎么问

```
我准备和一个潜在用户做访谈，验证我产品的需求。
请基于 The Mom Test 原则（不要问假设性问题，只问过去具体行为）帮我列访谈问题。

产品想法：[一句话]
被访者：[他是谁、和你什么关系]
我想验证的核心假设：[假设 1 / 假设 2 / 假设 3]

请给我：

1. **开场（2 分钟）** —— 暖场，引导用户进入状态
2. **过去行为（15 分钟）** —— 10 个问题，全部是"上次你 X 时是怎么做的""你 上周 X 次"
3. **痛点深挖（10 分钟）** —— 5 个问题，验证"这个痛是不是真的痛"
4. **价值观验证（5 分钟）** —— 不直接问"愿意付钱吗"，而是问"你为类似的 X 付过钱吗 / 多少钱"
5. **收尾（3 分钟）** —— "还有什么我应该问但没问的""能不能介绍我和 1-2 个相似的朋友聊"

每个问题用一句话，不要让我用引导性语言。

注意避免：
- "你会用 X 吗？" → 改成"你上次类似场景是怎么做的？"
- "你愿意每月付 $19 吗？" → 改成"你为类似工具付过钱吗？多少？"
- "这个产品好吗？" → 不要问观点，问行为
```

## P-15 · PRD prompt

**用途**：从产品想法 → 一份能扔给 AI / 工程师执行的 PRD

```
请帮我把下面这个想法写成一份完整 PRD（产品需求文档）。

产品想法：[一句话]
目标用户：[人群]
核心场景：[他们什么时候用]
我已有的约束：[预算 / 时间 / 技术栈]

PRD 结构：

# [产品名] PRD

## 1. 背景与目标
- 为什么做这个
- 解决什么问题
- 目标用户画像
- 成功指标（3 个，能量化）

## 2. 用户场景与故事
- 主场景 A：用户怎么发现 → 怎么用 → 用完后什么变化
- 主场景 B：…
- 主场景 C：…

## 3. 功能列表
按 MVP / V1 / V2 三阶段分：
- MVP 必须有：[5 个功能]
- V1 期待有：[5 个]
- V2 未来：[未定]

## 4. 关键 UI 描述
- 几个核心页面布局（文字描述）

## 5. 技术方案
- 技术栈
- 数据库 schema 概要
- 第三方服务（Stripe / Auth / 等）
- 部署平台

## 6. 里程碑
- Week 1：…
- Week 2：…
- Week 3：上线

## 7. 风险与未知
- 技术风险（3 个）
- 市场风险（3 个）
- 哪些假设需要验证

请写得能扔给 Cursor / Claude Code 让 AI 直接照着实现 MVP。
```

## 怎么用这些 prompts？

1. **打开你的 AI IDE / ChatGPT**
2. **挑一个 P-XX 复制**
3. **替换 `[方括号]` 里的内容**——这是你的项目特定信息
4. **粘进对话框 → 发送**
5. **追问**：AI 第一版通常 70 分。让它 "再迭代一版 / 加点 / 减点 / 改个风格"

**进阶**：把这些 prompt 存进 Cursor / Claude Code 的 Saved Prompts，或者写进 `.cursorrules` 让 AI 知道"遇到 X 任务时用 P-XX 风格"。
