# H-01 Product Hunt 发布

## 一句话定义
Product Hunt（PH）是**全球独立开发者"产品上线"的默认宣发平台**——每天全站有一个"当日最佳产品"排行榜，Upvote 高的产品会被推上首页、被投资人和媒体看到。**一次成功的 PH 发布 = 首日 1000-5000 个真实用户**。

## 打个比方
**像"独立开发者的奥斯卡"**：不是说你得奖才算成功，而是**你站上红毯，全球媒体和观众都在看**。一次精心策划的 PH 发布，能给你带来"别人花 $10K 广告费都买不到"的种子用户和媒体曝光。

## 和 vibe coding 的关系
- **Phase 6（宣发变现）的第一站**——你做完产品就该去 PH
- 要求：产品有可访问的页面 + 能演示核心功能（不一定要付费）
- 和 H-02（X BIP）、H-03（小红书）形成"PH→X 长尾讨论→国内社区"的宣发链

## 典型场景 / 示例

### 发布前 7 天准备清单

| 日期 | 做啥 |
|---|---|
| **D-7** | 在 PH 注册账号（Welcome）、完善头像+Bio+社交链接 |
| **D-7** | 写 5 版 tagline（PH 的 tagline 是"一句话卖点"，最多 60 字符） |
| **D-5** | 做 5 张产品截图 / GIF（PH 首页 5 张图是最重要的"第一印象"） |
| **D-5** | 注册上架（Schedule）：选一个**周二/周三/周四**（避开周末） |
| **D-3** | 在 X / Reddit / 即刻发预热帖："我下周在 PH 上线，先来波预告" |
| **D-3** | 拜托 10-20 个朋友/老用户：**不要在 PH 首页直接点 upvote**（会被算法标记刷票），而是让他们"自然搜索到你的产品、点进去、再 upvote" |
| **D-1** | 写"Maker Comment"（第一条评论，介绍你的产品、为什么做、怎么用、接下来计划） |
| **D-Day** | 在 PH 上线后 1 小时内发 Maker Comment + 回复所有用户评论 |
| **D-Day** | 全网渠道发帖："今天在 PH 上线，欢迎来玩"（每个渠道带 PH 链接） |
| **D+1** | 总结当日数据、回复剩余评论、写感谢信给 upvoter |

### 关键原则

1. **不要直接分享投票链接**——PH 算法会检测"外部直链投票"并把它们降权或无效化
2. **投票重量排序**：组织者 > 活跃用户 > 新用户 > 境外用户
3. **你的角色是"客服"**——上线日全天泡在 PH 上回复每条评论（越快越好）
4. **Maker Comment 比产品描述更重要**——用户看完截图先看 Maker Comment，那篇是你"讲故事"的唯一机会

### 什么产品 PH 不接

- 纯 API / 技术工具（没有可视化界面）→ 做成 demo 页面 + 截图
- 需要注册才能看到任何功能的 → 做一个"ghost user"免登录 demo
- 仅移动 App → 至少要有一个网页 Product Page

### 一个真实的 PH tagline 进化（vibe coding 风格）

```
❌ 差："AI 驱动的笔记应用"
✅ 好："Your AI-powered second brain that remembers everything you forget"
✅ 更好："The note-taking app that writes itself — AI organizes your chaos into clean notes"
```

## 常见误区
- ❌ **"PH 上线就是发个链接"**：**准备 7 天、上线 1 天、复盘 1 天**。没有准备就上 ≈ 白上。
- ❌ **"找朋友疯狂 upvote"**：会被算法 flag 为刷票，**反而拉低排名**。
- ❌ **"排名不重要"**：排名确实不重要——**重要的是"被 X/Reddit/媒体二次传播"**。PH 只是一根火柴，柴火要在其他渠道事先铺好。
- ❌ **"只发一次"**：可以多次上 PH（产品大更新、发布移动端、融到资等），但每次必须"有实质新东西"。
