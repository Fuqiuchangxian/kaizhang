# H-06 SaaS 定价策略

## 一句话定义
SaaS 定价 = **"用户愿意付多少" × "你成本多少" × "竞品收多少"** 三者的平衡——大多数独立开发者的核心误区是**定价太低**（怕没人买）而非太高。**做好的产品配得上更贵的价格**。

## 打个比方
**像开餐厅定菜价**：
- 太贵 → 没人进店
- 太便宜 → 客流大但利润薄，**且让顾客怀疑"是不是难吃"**
- 合理 → 让目标客群觉得"超值"，不让低端客群进来浪费你时间

## 和 vibe coding 的关系
- 大部分独立开发者一辈子在和"定价"较劲
- **从月费 $5 涨到 $19 比从 $0 涨到 $5 容易得多**——先定中位数
- 配合 H-07（找第一个用户）：低价获取早期用户，验证后调价

## 典型场景 / 示例

### 5 种主流定价模型

#### 1. 单层 / 简单 SaaS（推荐 0→1 阶段）
```
$19 / 月  · 所有功能
```
**优点**：决策简单、转化快
**适合**：单一产品 + 单一用户群

#### 2. 三档分级（Freemium / Pro / Team）
```
Free   $0     基础功能 + 限额
Pro    $19    无限额 + 高级功能
Team   $49    多人协作 + 管理
```
**优点**：覆盖不同付费意愿
**陷阱**：Free 档容易"白嫖率高、付费转化低"

#### 3. 按用量 / 按 Token
```
$0.001 / 千 token
预存 $20 / $100 / $500 套餐
```
**适合**：AI API 产品（成本可变）
**陷阱**：用户对账单不可控反感

#### 4. 一次性买断（lifetime deal / LTD）
```
$99 一次买断，终身使用
```
**用途**：早期收集种子用户（在 AppSumo 等平台）
**风险**：长期来看不可持续

#### 5. 个人版 / 企业版 二阶（B2C + B2B）
```
个人  $9 / 月
企业  $99 / 月（含 SSO / 审计 / 优先支持）
```
**适合**：to B 但也卖个人

### 6 条独立开发者定价心法

1. **永远不要免费**：哪怕 $5 / 月——付费用户的反馈远比免费用户有用
2. **价格里藏 9**：$9 / $19 / $29 / $49 / $99 / $199（行为经济学已验证）
3. **年付打折但保 7-10 折**：年付 $190（月付 $19）= 月付 $19 × 10 月。**别打 5 折，会让用户觉得你的产品"虚高"**
4. **早期 3 个月可以降价**：first 100 users 享受半价，未来涨价不补差——制造紧迫感
5. **复杂功能 = 高价**：能省用户 1 小时/周 = $20 / 月就值
6. **涨价比降价容易**：先定中位价 → 收集付费意愿 → 涨价。**降价等于"我之前在骗你"**

### 国内 vs 海外的价格锚定

| 用户群 | 心理锚 | 推荐 |
|---|---|---|
| **海外用户（USD）** | 同行 / Notion $10 / Linear $8 / Cursor $20 | $19-29 / 月 |
| **国内用户（CNY）** | 不是 $19 × 7 = ¥133；是国内 SaaS 锚（飞书 $0 起 / 钉钉 / 印象笔记 ¥148/年） | ¥29-99 / 月 |
| **同一产品两边卖** | 不要"汇率定价"——按各自市场锚 | USD 用 $19，CNY 用 ¥39 |

### Stripe / Creem / Lemon Squeezy 怎么实施？

详见 F-08。**关键**：定价改的时候在后端用"价格 ID"，不是改前端写死数字——历史用户保留旧价，新用户用新价。

## 常见误区
- ❌ **"先免费引流，再涨价"**：90% 失败。**免费用户极少转付费**。直接收费 + 退款保证 比 freemium 转化率高。
- ❌ **"做得多 = 该多收钱"**：用户付的是"对他们的价值"，不是"你写了多少代码"。
- ❌ **"我比竞品便宜就会赢"**：价格战是死路。**用户买"对他来说值不值"，不是"比竞品便宜多少"**。
- ❌ **"折扣码越多越好"**：折扣码会降低未来用户的"愿付价"心理预期。**少用 + 有正当理由**（黑五 / 创始人折扣 / 上线限时）。
- ❌ **"按月付费一定是 SaaS"**：单价 / 一次性 / 按用量都能成 SaaS。**适合自己产品的就是好定价**。
