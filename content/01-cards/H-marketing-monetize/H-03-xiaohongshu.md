# H-03 小红书宣发

## 一句话定义
**小红书** = 国内 vibe coder 触达"非技术中文用户"的最佳渠道——**用户群以 25-40 岁女性 + 高学历高消费人群为主**，对 AI 工具、效率工具、独立小众产品接受度极高。

## 打个比方
**像中国版 Pinterest + 大众点评**：用户来这里**搜索"X 怎么做"**（不是刷信息流），所以你的产品如果能解决某个具体痛点，被搜到的概率比 X / 即刻高 10 倍。**"搜索流量"才是小红书最大的红利**。

## 和 vibe coding 的关系
- **国内中文产品最重要的宣发渠道**（其次是即刻 H-05）
- 适合产品：AI 工具 / 效率工具 / 学习工具 / 设计工具 / 生活工具
- 一篇爆款笔记（1 万赞）= 大约 10-50 个种子用户
- 长尾效应强：发布 3-6 个月后仍持续带流量

## 典型场景 / 示例

### 笔记类型 · 5 个最有效模板

#### 1. 痛点解决型（最经典）
```
标题：再也不用每天手动整理日报了 ｜ AI 给我打工
正文：
"作为一个'摸鱼'打工人，最痛苦的就是周报日报...
直到我自己做了个 AI 工具帮我自动写..."
[3 张截图]
"现在每天 3 分钟搞定，还得到老板表扬 hhh"
```

#### 2. 教程型（搜索流量）
```
标题：手把手教你 30 分钟做出第一个 AI 应用｜0 基础
正文：分步骤教程 + 截图 + 工具链接
```

#### 3. 对比型（强种草）
```
标题：试了 5 款 AI 笔记工具，最后留下了这 2 个
正文：表格 + 优缺点 + 推荐人群
```

#### 4. 数据/成绩型
```
标题：副业一个月做了一个 AI 工具，今天卖出第一份订阅
正文：MRR 截图 + 复盘
```

#### 5. 反向种草型
```
标题：xxx 这些 AI 工具千万别买
正文：避雷 + 推荐替代（顺手安利你自己产品）
```

### 标题黄金公式

```
[痛点] + [数字] + [反差/钩子] + [emoji]

例子：
- "副业 0 基础做 AI 工具，月入 5 位数的真实复盘 💼"
- "30 天我用 Cursor 做了 3 个产品，最后只活下来 1 个"
- "再也不熬夜改 PPT 了｜AI 帮我 5 分钟搞定"
```

### 内容创作 4 个不能省的细节

1. **封面图**：1080×1440 竖图，**首屏 3 个字必须看清重点**（用大字 + 强对比色）
2. **第一行正文** = 钩子：决定用户是否继续看
3. **配图 6-9 张**：第一张是封面、后面是流程截图、最后一张是 CTA
4. **位置 + 话题**：发布时加 3-5 个相关 #话题（#AI工具 #独立开发 #效率工具 等）

### 软广边界

小红书对硬广敏感。**只在帖子末尾或评论区"轻轻"放产品链接**——例如：
> "工具叫 xxx，私信回'AI'获取 demo"
> "工具我放在评论区了，自取" + 评论：xxx.com

**绝对不要**：① 标题直接写产品名 ② 全文 5 次以上提产品 ③ 删除负面评论。**会被限流**。

### 多账号 + 矩阵

策略型独立开发者会用 3-5 个不同人设的账号（实习生视角 / 宝妈视角 / 大学生视角 / 设计师视角）**同一个产品多角度种草**——但这是灰色地带，新手谨慎。

## 常见误区
- ❌ **"小红书 = 给 18 岁女生用"**：错。25-40 岁高消费人群占主流，男性用户也持续增长。
- ❌ **"发一篇就能爆"**：单条爆款率 < 1%。**坚持发 20-50 篇 + 不停 A/B 测试标题封面**，才有概率出爆款。
- ❌ **"小红书算法不重要"**：第一小时的"完读率 + 互动率"决定了能不能进流量池。**精心做第一篇，比海量水帖有效**。
- ❌ **"评论区是 vibe coder 主场"**：恰好相反——**小红书评论区氛围更口语**，回复太"技术男"反而失分。学着用"姐妹 / 宝子 / 真的吗" 等中性日常语气。
