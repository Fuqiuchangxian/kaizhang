# H-08 变现案例库

## 一句话定义
这张卡是"**学习他人 vibe coding 变现路径**" 的资源导航——独立开发者最该读的不是教程，而是**别人的真实收入复盘 + 失败教训**。

## 打个比方
**像学做菜读"米其林餐厅老板访谈"**：技巧、菜谱、设备这些都可以学，但**"老板真实开店的踩坑日记"是教不出来的隐性知识**——只能从案例里看。

## 和 vibe coding 的关系
- 看 100 篇案例 > 看 1000 个 vibe coding 教程
- 通过案例你能看到："这个人怎么做的、什么时候 PMF、定价怎么调、第一波用户哪来"
- 这些**模式比任何 framework 都重要**

## 怎么读案例（4 步法）

1. **看产品**：他做的是什么、解决谁的痛点
2. **看路径**：他怎么找到第一个用户？冷启动用了多久？
3. **看数据**：MRR 增长曲线长什么样？多久达到 $1K / $5K / $10K MRR？
4. **看转折**：他踩过什么坑？哪个决定让 MRR 翻倍？哪个让他差点放弃？

读完每个案例做笔记：**"我能学的一条 + 我能避的一坑"**。

## 常见误区
- ❌ **"复制别人的案例就能成功"**：错。**复制路径但要解决你自己擅长的领域**。
- ❌ **"看了 100 个案例就够了"**：不够。看完去做，做了再回来看，循环 N 轮。
- ❌ **"国内做不到海外那种 MRR"**：错。国内做 to B + 私域 + 出海，**已经有月入万元的独立开发者**。只是公开度低。
- ❌ **"案例都是幸存者偏差"**：是。**但你需要的不是"统计学结论"，是"模式 / 工具 / 心态"**。
