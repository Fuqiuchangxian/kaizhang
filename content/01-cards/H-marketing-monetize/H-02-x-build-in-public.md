# H-02 X（Build in Public）

## 一句话定义
**Build in Public（BIP）** = 在做产品的过程中**公开分享一切**——idea、进度、数据、收入、失败教训——把"做产品的过程"本身变成内容。X（原 Twitter）是英文圈 BIP 的事实大本营。

## 打个比方
**像直播间盖房子**：你不在房子盖好那天剪彩，**而是从挖第一锹土开始就开直播**——围观群众跟着每天看进度、出主意、加油。等房子建好那天，你已经积累了**几百上千个真实关心你的潜在用户**——他们更愿意付费、更愿意宣传。

## 和 vibe coding 的关系
- **0 用户 → 1000 真实关注者**最有效的路径之一（不需要广告预算）
- 适合"做"的部分多于"营销"的人——你只要把每天的真实进度发出来即可
- 一个 BIP 账号通常 3-12 个月能积累到 1k-10k followers
- 配合 PH 发布（H-01）效果更佳：BIP 期间累计的粉丝是 PH 上线日最忠诚的 upvoter

## 典型场景 / 示例

### BIP 内容应该发什么

| 类型 | 例子 | 频率 |
|---|---|---|
| **进度** | "今天加了 Stripe 集成，从 0 到付费用户体验只要 30 秒" | 每天 |
| **数据** | "MRR 截图 / DAU 截图 / 第一个付费用户截图" | 每周 |
| **失败/学到** | "今天部署炸了 2 小时，原因是忘了配 envvar，复盘如下" | 每周 |
| **疑问** | "你们用 X 做 Y 时是怎么处理 Z 的？" | 不定期 |
| **预告** | "下周一上 PH，先来一波 sneak peek" | 节点 |
| **复盘** | "30 天独立开发月度总结" | 月度 |

### BIP 的副产品
- **粉丝 → 用户转化率**：BIP 粉丝是冷流量的 **20-50 倍**忠诚
- **媒体接触**：积累几千粉丝后会有 podcast / blog 邀请采访
- **同行支持**：BIP 圈互相 RT / 帮忙测试 / 给反馈，社区效应强

中文圈的 BIP 阵地主要在**即刻**和**小红书**（H-03）。X 上中文 BIP 圈仍偏小但在长。

## 常见误区
- ❌ **"BIP = 晒成功"**：恰相反——**失败和教训更受欢迎**。一篇"复盘 3 个月没卖出去一单"的帖子常比"上线第一天 MRR $100"还火。
- ❌ **"必须每天发"**：质量 > 频率。一周 3-5 条高质量贴 > 每天发流水账。
- ❌ **"等做完再开始"**：BIP 的核心是**做的过程公开**。等做完才发等于普通宣发。
- ❌ **"中文 BIP 没人看"**：错。即刻、小红书的中文 BIP 圈正在长，竞争反而比 X 少。
- ❌ **"BIP 必须晒 MRR"**：很多人对收入保密。**晒进度 / 学到 / 失败都行**——重点是真实、有干货。
